# PRD：美股涨了，A股呢？

> 版本：MVP v2.0 | 2026-04-11
> 状态：待评审
> 基于 v1.0 + 花叔视角评审反馈精简

---

## 1. 一句话说清楚

**美股昨晚涨了，今天 A 股跟不跟？** 一句话说清楚概率，30 秒做完判断。

**不是什么**：不推荐个股、不预测价格、不构成投资建议。所有输出均为历史统计数据。

---

## 2. 目标用户与场景

**用户**：A股散户，关注美股隔夜走势对 A 股开盘的影响。

**场景**：每天早上 8:30-9:25，打开 H5 → 看到几张卡片 → "半导体昨晚涨 4%，A 股半导体 ETF 今天 78% 概率高开" → 决定要不要关注这只 ETF。

**不是场景**：学术研究、量化策略、个股选择。

---

## 3. MVP 范围（2 天）

### 3.1 板块映射（5 对）

选相关性最高、散户最关注的 5 对：

| # | 美股板块 | 美股 ETF | A 股板块 | 申万指数 | A 股 ETF |
|---|---------|---------|---------|---------|---------|
| 1 | 半导体 | SOXX | 半导体 | 801081 | 半导体ETF(512480) |
| 2 | 科技 | XLK | 电子 | 801080 | 科技ETF(515000) |
| 3 | 新能源车 | DRIV | 汽车 | 801880 | 新能源车ETF(515030) |
| 4 | AI/算力 | THNQ | 计算机 | 801750 | 计算机ETF(512720) |
| 5 | 黄金 | GLD | 有色金属 | 801050 | 黄金ETF(518880) |

### 3.2 统计指标（1 套）

只算 60 日窗口，只看 2 个指标：

| 指标 | 说明 | 用户看到的 |
|------|------|-----------|
| 条件概率 | 60 日内，美股板块涨跌 > 2% 时，A 股板块次日高开（> 0%）的概率 | "78% 概率高开" |
| 平均幅度 | 条件触发后 A 股板块的平均涨跌幅 | "平均高开 0.8%" |

显著波动阈值：美股板块单日涨跌幅绝对值 > 2%。

### 3.3 页面（1 个，只有卡片）

```

```

**规则**：
- 只展示涨跌幅 > 2% 的板块为"显著波动"卡片，按条件概率降序
- 其余板块折叠在"今日无显著波动"下方，只显示涨跌幅
- 涨红跌绿（A 股习惯）
- 最大宽度 480px，居中
- 纯静态 JSON，无实时刷新

### 3.4 自动化

GitHub Actions 每个交易日北京时间 05:00 自动执行：
1. yfinance 拉取 5 个美股 ETF 数据
2. AKShare 拉取 5 个申万指数 + 5 个 A 股 ETF 数据
3. 计算 60 日条件概率 + 平均幅度
4. 输出 JSON，git push
5. GitHub Pages 自动部署

## 4. 传播策略

**没有传播的产品不是产品。**

| 渠道 | 内容 | 频率 | 时间 |
|------|------|------|------|
| **小红书**（主战场） | "美股半导体昨夜涨 4%，A 股半导体 78% 概率高开" + H5 链接截图 | 每个交易日 | 早上 8:30 |
| 微信群/朋友圈 | 同上，简短版 | 每个交易日 | 早上 8:30 |

**标题公式**：`"美股{板块}昨夜涨{X}%，A股{ETF}今日{概率}%高开"`

**内容模板**：
1. 一张卡片截图（视觉冲击）
2. 一句话结论（数字说话）
3. H5 链接（转化入口）
4. 免责声明（合规）

## 5. 技术方案

| 层 | 选择 | 理由 |
|---|------|------|
| 美股数据 | yfinance v1.2.0+ | 免费，`yf.download()` 一次拉 5 个 ETF |
| A 股数据 | AKShare | 免费，`sw_index_daily` + `fund_etf_hist_em` |
| 统计计算 | pandas + numpy | 标准 Python 生态 |
| 存储 | JSON 文件 | 每日 ~10KB |
| 定时任务 | GitHub Actions | 公开仓库免费无限制 |
| 前端 | 原生 HTML/CSS/JS（Vite） | 5 个卡片，不需要图表库 |
| 部署 | GitHub Pages | 零成本 |

**项目结构**：
```
US-CN-Sector-Mapper/
├── data/
│   └── results/YYYY-MM-DD.json
├── scripts/
│   └── run_daily.py           # 数据拉取 + 计算 + 输出（单文件）
├── web/
│   ├── index.html
│   ├── src/main.js
│   └── src/style.css
├── .github/workflows/daily_update.yml
└── requirements.txt
```

## 6. 验收标准

- [ ] 能拉取 5 对板块最近 60 个交易日数据
- [ ] 条件概率计算正确（抽样 2 对验证）
- [ ] H5 在 iPhone SE（375px）正常展示
- [ ] 涨跌幅 > 2% 正确标记为显著波动
- [ ] 卡片展示关联 A 股 ETF 代码
- [ ] 含免责声明
- [ ] GitHub Actions 每日自动执行
- [ ] 页面首屏 < 3 秒

## 7. MVP 不做（V2 再说）

| 功能 | 为什么不做 |
|------|----------|
| 15 对板块 | 先验证 5 对有没有人看 |
| 30/120 日滚动窗口 | 60 日够用，多窗口增加认知负担 |
| 热力图 | 开发成本高，MVP 不需要 |
| 完整对照表 | 没人会在手机上看 15 行表格 |
| 日期选择器 | 先只看最新一天 |
| 港股对标 | 聚焦 |
| 推送通知 | 等有用户再说 |
| ECharts | 5 个卡片不需要图表库 |
| 申万指数同时展示 | 只展示 ETF，用户关心的是可交易标的 |

## 8. 风险

| 风险 | 缓解 |
|------|------|
| AKShare 接口失效 | 锁版本；备选 baostock |
| 没人用 | 2 天做完，沉没成本低；小红书验证需求 |
| 法律边界 | 措辞严格中立；完整免责声明 |

## 9. V2 路线图（MVP 验证后）

1. **扩板块**：5 → 15 对，根据 MVP 数据看哪对相关性最强
2. **加热力图**：用户需要全局视角时再加
3. **多窗口**：加入 30/120 日对比
4. **推送通知**：微信公众号或小红书自动发布
5. **港股对标**：第二市场
